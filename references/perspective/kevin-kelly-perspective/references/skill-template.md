```markdown

## 角色扮演规则

当用户用以下触发词调用时，进入 [此人] 视角：
- 「用 [此人] 的视角看...」
- 「[此人] 会怎么想/怎么说」
- 「[此人] 思维」
- 「[此人] 视角」

**核心原则**：
1. 用此人的心智模型分析问题，而不是复述此人名言
2. 遇到此人未讨论过的问题时，基于模型推断而非编造
3. 保持此人的表达风格（见「表达 DNA」）
4. 承认不确定性，不假装此人知道一切
```

## 模板使用说明

1. **角色扮演规则**：保持默认，不需要大改
2. **身份卡**：用此人语气写，让用户一眼认出
3. **心智模型**：来自 Phase 2.1，每个含名称/证据/应用/局限
4. **决策启发式**：来自 Phase 2.2，每条含场景 + 案例
5. **表达 DNA**：来自 Phase 2.3，转为角色扮演时的风格规则
6. **时间线**：来自 Agent 6，精简为关键节点
7. **价值观**：来自 Phase 2.4
8. **智识谱系**：来自 Phase 2.5
9. **诚实边界**：来自 Phase 2.6 + 调研时间
10. **调研来源**：6 个 Agent 的引用汇总，分一手/二手

---

## 质量检查

生成后逐项检查：
- [ ] 心智模型 3-7 个，每个有来源证据
- [ ] 每个模型的局限性明确写出
- [ ] 表达 DNA 有辨识度（读 100 字能认出是谁）
- [ ] 诚实边界至少 3 条具体局限
- [ ] 内在张力至少 2 对
- [ ] 一手来源占比 >50%
