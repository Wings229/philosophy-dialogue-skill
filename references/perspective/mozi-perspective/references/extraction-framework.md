# 心智模型提炼框架

> 用于从调研素材中提取真正独特的心智模型，而非通用道理。

---

## 三重验证法

一个论点要成为「心智模型」，必须通过以下三重验证：

### 1. 跨域复现 (Cross-Domain Repetition)
- 此论点在 ≥2 个不同领域/话题中出现？
- 例如：KK 的「演化论」视角同时出现在技术、生物、经济讨论中

**操作**：扫描 01-writings.md 到 05-decisions.md，标记每个论点出现的场景数量。

### 2. 生成力 (Generative Power)
- 用此论点能否推断此人对新问题的立场？
- 例如：用庄子的「齐物」可以推断他对 AI 伦理问题可能的态度

**操作**：选 3 个此人未公开讨论的问题，用候选模型推断，看推断是否合理。

### 3. 排他性 (Exclusivity)
- 这个想法是此人独特的，还是「所有聪明人都这样想」？
- 例如：「要诚实」是通用道理，「1000 True Fans」是 KK 独有的

**操作**：问自己「这个观点能不能安到别人头上？」如果不能，排他性高。

---

## 筛选流程

```
候选论点池 (15-30 个)
    ↓
三重验证筛选
    ↓
├── 3 重通过 → 心智模型 (目标 3-7 个)
├── 1-2 重通过 → 决策启发式 (目标 5-10 条)
└── 0 重通过 → 丢弃
```

## 心智模型记录格式

```markdown
### [模型名称]

**一句话描述**：[此模型的核心洞察]

**来源证据**：
- [场景 1]：[出处，如《必然》p.45 / Tim Ferriss 播客 2019]
- [场景 2]：[出处]
- [场景 3]：[出处]

**应用方式**：[如何用此模型分析问题]

**局限性**：[此模型失效的条件]
```

---

## 决策启发式记录格式

```markdown
### [启发式名称]

**规则**：如果 [条件 X]，则 [行动 Y]

**案例**：[具体决策案例，如「2005 年拒绝某收购因为...」]

**来源**：[出处]
```

## 表达 DNA 分析维度

| 维度 | 问题 |
|------|------|
| 句式偏好 | 长句/短句？疑问/陈述？类比密度？ |
| 词汇特征 | 高频词？专属术语？禁忌词？ |
| 节奏感 | 先结论还是先铺垫？转折方式？ |
| 幽默方式 | 讽刺/自嘲/荒诞/冷幽默/不幽默？ |
| 确定性表达 | 「我不确定」型 还是 「很明显」型？ |
| 引用习惯 | 爱引谁？引什么类型？ |

## 质量自检清单

完成提炼后逐项检查：

- [ ] 心智模型数量在 3-7 个之间
- [ ] 每个模型都有 ≥2 个场景的来源证据
- [ ] 每个模型都明确写出了局限性
- [ ] 决策启发式有具体案例支撑
- [ ] 表达 DNA 能让陌生人读 100 字认出是谁
- [ ] 至少记录了 2 对内在张力/矛盾
- [ ] 一手来源占比 >50%
- [ ] 没有编造此人没说过的话
- [ ] 没有把通用道理包装成「独特见解」

---

## 常见陷阱

| 陷阱 | 信号 | 修正 |
|------|------|------|
| 过度概括 | 「他总是...」「他从不...」 | 回到原始材料，找反例 |
| 金句拼凑 | 模型 = 此人名言集锦 | 问：这揭示了什么思维结构？ |
| 忽略矛盾 | 所有观点高度一致 | 矛盾是深度的来源，保留它 |
| 二手污染 | 主要来源是书评/转述 | 回到一手材料验证 |
| 时代错位 | 把晚年观点当早期 | 用时间线校正 |

---

**最后提醒**：宁要 3 个深刻的模型，不要 10 个浅薄的原则。
